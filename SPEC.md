## 1. システム構成

```
imstest/
└── index.html   # フォーム本体（HTML + CSS + JavaScript）
```

- **フロントエンド:** 静的HTML（フレームワーク不使用）
- **ホスティング:** GitHub Pages（main ブランチ ルート `/`）
- **フォーム送信:** SSGform（外部フォームバックエンドサービス）
- **課題登録先:** Backlog（SSGform → メール → Backlog 自動登録）

### 送信フロー

```
ユーザー入力
    ↓
JavaScript でバリデーション・メール本文を整形
    ↓
報告者に対応する SSGform URL を動的に決定
    ↓
form.submit() で SSGform へ POST
    ↓
SSGform がメール送信
    ↓
Backlog メール受信 → インシデント課題として自動登録
```

### A セクション：基本情報（発生日時・場所・対象機器）

|-----------|------|:----:|------|

| 号機 | テキスト | ✓ | 例：#03、#07 |
| エラーコード / 種別 | テキスト | | 例：E-204、通信断 |

### B セクション：優先度・影響範囲（影響度と緊急度の評価）

| フィールド | 種別 | 必須 | 選択肢 |
|-----------|------|:----:|--------|
| 優先度 | クリックボタン（P1〜P4） | ✓ | P1（緊急）/ P2（高）/ P3（中）/ P4（低） |
| 影響範囲 | セレクトボックス | ✓ | 生産停止（全ライン）/ 生産停止（一部ライン）/ 速度低下・遅延 / 代替手段で運用中 / 影響なし |
| 運用継続可否 | セレクトボックス | ✓ | 全機能使用不可 / 一部機能のみ使用不可 / 代替手段で運用可能 / 通常運用に支障なし |
| 対応期限（目標） | datetime-local | | SLAまたは顧客要求に基づく設定 |

#### 優先度ボタン カラー定義

| 優先度 | 表示色 | 用途の目安 |
|--------|--------|-----------|
| P1 緊急 | 赤 | 生産停止・即時対応が必要 |
| P2 高 | オレンジ | 業務に重大な影響あり |
| P3 中 | 青 | 一部影響あり・代替運用中 |
| P4 低 | 紫 | 軽微・監視継続で対応可 |

## 3. ボタン機能

| ボタン | 動作 |
|--------|------|
| 書き出し保存 | フォーム内容を整形したテキストファイル（.txt）をダウンロード、かつ localStorage に下書き保存 |
| 送信 | 入力検証 → メール本文を整形 → 報告者に対応する SSGform URL へ POST 送信 |

### 下書き復元

- ページ読み込み時に localStorage に保存データが存在する場合、復元確認ダイアログを表示
- 確認後、全フィールドと優先度ボタンの選択状態を復元

送信先URLは JavaScript の `SSGFORM_URLS` オブジェクトで管理しており、送信ボタン押下時に報告者の選択値をキーとして URL を決定し `form.submit()` で送信する。

### POSTフィールド

送信時に `name` 属性を持つ全フィールドが POST される（合計25フィールド）。

| フィールド名 | 内容 |
|-------------|------|
| `userName` | 報告者 |
| `occurrenceTime` | 発生日時 |
| `reportTime` | 報告日時 |
| `location` | 発生場所 |
| `frequency` | 発生頻度 |
| `model` | 機種 |
| `unitNumber` | 号機 |
| `errorCode` | エラーコード / 種別 |
| `priority` | 優先度（hidden、JSがセット） |
| `impactScope` | 影響範囲 |
| `operationStatus` | 運用継続可否 |
| `deadline` | 対応期限 |
| `when5w` | When |
| `where5w` | Where |
| `who5w` | Who |
| `what5w` | What |
| `why5w` | Why |
| `detail` | 事象の詳細 |
| `investigation` | 調査内容 |
| `reproducibility` | 再現性 |
| `knownError` | 既知エラーとの一致 |
| `actionRequest` | 依頼内容・アクション |
| `notes` | その他・特記事項 |
| `subject` | メール件名（hidden、JSが生成） |
| `message` | メール本文・整形済み全文（hidden、JSが生成） |

### メール件名フォーマット

```
[インシデント報告] {機種} {号機} - {優先度} ({発生日時}) [{報告者}]
```

例：
```
[インシデント報告] WILL-FA #03 - P1 (2026-04-22 09:32) [Rapidus A/W]
```

## 5. バリデーション

| チェック内容 | タイミング | 動作 |
|-------------|-----------|------|
| 報告者未選択 | 送信ボタン押下 | トースト通知を表示し送信中断 |

## 6. 変更・拡張時の注意点

- **報告者の追加・変更:** `index.html` 内の `<select id="userName">` の `<option>` と `SSGFORM_URLS` オブジェクトを両方編集する
- **機種の追加・変更:** `<select id="model">` の `<option>` を編集する
- **SSGform送信先変更:** `SSGFORM_URLS` オブジェクトの該当URLを変更する
- **デプロイ確認:** ページ最下部のフッターにバージョン日付（`ver. YYYY-MM-DD`）を表示。コード変更時は日付を更新する
- **GitHub Pages 反映:** `main` ブランチへプッシュ後、数分で自動反映される

## 7. デプロイ手順

```bash
# リポジトリクローン
git clone https://github.com/YoshiNabe/imstest.git
cd imstest

# 編集後にプッシュ（フッターのバージョン日付も更新すること）
git add index.html
git commit -m "変更内容のコメント"
git push origin main
```

GitHub Pages は `main` ブランチを監視しており、プッシュ後数分で https://yoshinabe.github.io/imstest/ に反映される。
